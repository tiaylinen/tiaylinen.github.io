MY PROJECTS

Prototyping in Unity

I built this prototype to demonstrate features and ideas for a team project. All concepts have been bundled up to a single scene, but should be separated for the final game. First, there is a falling floors level. In this, the floor tiles start falling off as the player collides with them and everyone has to try to survive as long as possible. Next up there are some power ups: both positive (green) and negative (red). The power ups in this prototype can either increase the size or speed of the player, or stop them from moving completely. Lastly, there's some heavy moving obstacles that will knock they player off the platform when collided with.

C++ game with physics and AI

This game was created as coursework around a codebase we were offered. When the game is started, the player starts competing against time or against an AI opponent. The player loses points over time and the final score is based on how much time there is left when the goal is reached. Using C++ I implemented the gameplay features and physics simulations, such as different types of collision detection and resolution, forces, position constraints and obstacles of different weights and behaviour. In addition to this implemented a simple state machine AI the player can choose to play against.

Graphics

This graphics coursework was created using C++ and OpenGL. The scene displayed shows a cityscape in a bumpy terrain with a lake outside the city. I created this scene around a codebase and added the textures and bump maps, moving water, the height map to generate the terrain, the floating and moving triangles, the city and the sky box. The scene uses appropriate shaders to render the implemented features. All geometry is added to a scene graph for efficiency and practicality. The program also comes with camera tracks to guide the user around the scene.

Maze generator & solver with collaborative pathfinding

This C++ program consists of two parts: first one to generate mazes and find the exits, and the second part for collaborative pathfinding to move players towards the center of the maze in turns. The user gets to decide the size of the maze and the number of exits/players. The program uses BFS to find the shortest paths. All mazes are generated so that they are solvable and each exit can be reached. In collaborative pathfinding each player takes a step towards the goal in turns. If another player gets in the way, that turn is skipped. In the end the user can choose to save a maze to a file or display a maze that has already been saved.

Quiz web app

This web app was created using HTML + CSS + JavaScript with React. The user can choose a quiz, answer the questions and see results in the end. For accessibility the quiz can be navigated through both with mouse or with arrows / tab / enter. After taking a quiz the user can add a comment that other users can see. The comment section is unique to each quiz. The results are calculated simply by which type of answer the user picks most of (a, b, c or d). The quizzes and comments are saved to a JSON server which the app communicates with using Axios.
